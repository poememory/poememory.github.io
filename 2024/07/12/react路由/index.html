react路由

路由是个啥？

前端路由是指在单页面应用中，通过改变浏览器的URL来动态加载不同的组件或视图，而无需刷新整个页面。后端也有路由的概念，后端路由是指在服务器端，将不同的URL请求映射到相应的处理函数或控制器方法上。

react路由用法

路由注册

修改入口文件，首先要做的是创建浏览器路由并配置我们的第一个路由。这将为我们的 Web应用启用客户端路由。

子路由嵌套

既然说前端路由是动态加载不同的页面与组件，当然避免不了组件的嵌套。

Root组件中在哪里渲染子组件捏？

就是这个Outlet组件，它会渲染路由注册中的子组件。

路由跳转

  1. 链接式，<Link to={/path}>点击跳转，可以先理解为<a href={/path}>，两者的区别后面再说。
  2. 编程式，事件触发后navigate跳转

路由传参

  1. useParams钩子，返回一个由 <Route path> 匹配的当前 URL 动态参数的键/值对组成的对象。子路由继承父路由的所有参数。
  2. searchParams传参，参数的形式以 问号拼接 到地址后面
  3. state传参

路由配置

  1. loader,每个路由都可以定义一个 loader函数，以便在路由元素渲染前为其提供数据。当用户在应用程序中导航时，下一个匹配路由分支的加载器将被并行调用，其数据将通过useLoaderData提供给组件。
  2. errorElement,当 loader、action 或组件渲染中出现异常时，路由的正常渲染路径 <Route element> 将被渲染为错误路径 <Route errorElement>，错误信息将通过 useRouteError显示。

杂谈

先来看看一个问题，react路由项目中如果一个根路径下有一个/pageA路由，而且后端也写了一个/pageA路由，此时在浏览器导航栏中输入http://..../pageA，前后端的路由各自怎么反应，前端会渲染/pageA组件吗，会发起请求吗？




首先我们明确一个问题，react应用其实在html中只有一个root元素，所有的内容都是挂载在root元素下的，也就是说我们其实一直在一个html页面中，第一次请求后所有的组件都已经在浏览器中了，不需要跳转页面。react路由会改变导航栏，浏览器的导航栏变化确实会发起对应的GET请求，但是react路由会监听导航栏变化阻止这个请求。因此react路由引起的导航栏改变不会发出对应请求。

原生请求时，会导致强制刷新，相当于重新请求一次后端，但使用react-router就可以减少请求。后面会详细讲解

react路由允许我们的应用程序更新 URL，而无需从服务器请求另一个文档。相反，应用程序可以立即呈现新的用户界面。

再看看我们上面的留下的a标签和Link的区别，a标签会发起原生请求，而Link就是发起react路由跳转

React Router 允许我们的应用程序更新 URL，而无需从服务器请求另一个文档。相反，应用程序可以立即呈现新的用户界面。与使用原生 a标签相比，React Router 的 Link 组件可以减少网络请求，提升用户体验。
